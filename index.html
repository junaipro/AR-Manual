🏠 홈 › 2. 업무별 처리방법 › 2.1 고객정보 관리

🏢 고객정보 관리

Oracle EBS에서 고객을 신규 등록하고 수금 조건, 세금 정보를 설정하는 방법을 안내합니다.

AR› 고객› 고객 표준 입력
📝
신규 고객 등록 절차
고객 마스터 생성 순서
  • 1

    고객 기본 정보 입력

    고객명, 고객 번호(자동/수동 선택), 사업자등록번호, 고객 유형(기업/개인)을 입력합니다. 고객명은 중복 체크를 반드시 실행하여 중복 등록을 방지합니다.

  • 2

    주소 정보 등록

    청구 주소(Bill-To)와 배송 주소(Ship-To)를 각각 입력합니다. 청구 주소는 인보이스 발송에 사용되므로 정확히 입력해야 합니다. 복수의 주소 등록이 가능합니다.

  • 3

    수금 조건 설정 (Payment Terms)

    결제 조건(Net 30, Net 60 등)을 지정합니다. 수금 조건은 인보이스 만기일 자동 계산에 사용됩니다. 사전에 등록된 결제 조건 중에서 선택합니다.

  • 4

    세금 정보 입력

    세금 코드, 과세 유형(과세/면세/영세율)을 선택합니다. 해외 거래 고객의 경우 VAT 등록 번호를 입력합니다.

  • 5

    신용 한도 설정

    고객별 신용 한도 금액을 입력합니다. 설정된 한도 초과 시 오더 입력 단계에서 경고 또는 보류 처리됩니다.

  • 6

    저장 및 검토

    모든 정보 입력 후 [저장] 버튼을 클릭합니다. 저장 후 자동으로 고객 번호가 채번됩니다.

📋
고객 등록 주요 필드 설명
필드명필수여부설명 및 주의사항
고객명 (Customer Name)필수법인명 또는 개인명. 공식 사업자 명칭과 일치시켜야 합니다.
고객 번호 (Customer Number)자동채번시스템 자동 생성. 수동 입력은 설정에 따라 다릅니다.
고객 유형 (Customer Type)필수Organization(기업) / Person(개인) 중 선택합니다.
수금 조건 (Payment Terms)필수Net 30, Net 60 등 사전 등록된 조건 중 선택합니다.
세금 코드 (Tax Code)선택해당 고객의 세율 코드. 미입력 시 인보이스에서 수동 입력해야 합니다.
신용 한도 (Credit Limit)선택미입력 시 한도 없음으로 처리됩니다.
⚠️
주의 고객 등록 후 사업자등록번호는 수정이 불가할 수 있습니다. 초기 입력 시 반드시 정확히 입력하세요. 수정이 필요한 경우 시스템 관리자에게 문의하세요.
🏠 홈 › 2. 업무별 처리방법 › 2.2 매출 전표 처리

📄 매출 전표(인보이스) 처리

Oracle EBS에서 매출 인보이스를 생성하고 승인, 전기 처리하는 방법을 안내합니다.

AR› 전표› 인보이스 입력
➕
인보이스 신규 입력
수동 인보이스 생성 절차
  • 1

    인보이스 유형 선택

    Invoice(일반 매출), Credit Memo(크레딧), Debit Memo(추가청구) 중 선택합니다. 인보이스 유형에 따라 GL 계정이 달라집니다.

  • 2

    고객 및 기본 정보 입력

    고객명 또는 고객 번호로 검색하여 선택합니다. 청구지 주소(Bill-To)를 확인하고, 인보이스 일자와 GL Date를 입력합니다. GL Date는 회계기간이 열려있는 날짜여야 합니다.

  • 3

    수금 조건 확인

    고객 마스터에서 기본 Payment Terms가 자동 불러와집니다. 건별로 다른 조건 적용이 필요하면 여기서 변경 가능합니다.

  • 4

    인보이스 라인 입력

    품목/서비스 설명, 수량, 단가, 세금 코드를 라인별로 입력합니다. 복수의 라인 입력이 가능하며, 라인별로 상이한 세율 적용이 가능합니다.

  • 5

    회계 정보 확인

    [회계 배분] 탭에서 차변/대변 계정을 확인합니다. 자동으로 생성된 계정 배분이 올바른지 검토합니다. (매출채권 차변 / 매출 대변)

  • 6

    인보이스 완료 처리

    검토 완료 후 [완료] 버튼을 클릭합니다. 완료 상태가 되어야 수금 적용 및 전기 처리가 가능합니다.

🖨️
인보이스 출력 및 전송
✅
인보이스 프린트 방법 인보이스 조회 화면에서 해당 인보이스 선택 후 [Actions] → [Print] 메뉴를 클릭합니다. 출력 형식(PDF/엑셀)을 선택하고 실행합니다.
📧
이메일 자동 발송 설정 고객 마스터에 이메일 주소가 등록된 경우, 인보이스 완료 시 자동 이메일 발송이 가능합니다. 설정은 시스템 관리자에게 문의하세요.
⚡
자동 인보이스 생성 (AutoInvoice)

Oracle EBS AutoInvoice 프로그램을 사용하면 외부 시스템(영업, 물류)에서 생성된 데이터를 AR 인보이스로 자동 변환할 수 있습니다.

  • 1

    인터페이스 테이블 데이터 적재

    RA_INTERFACE_LINES_ALL 등 AR 인터페이스 테이블에 인보이스 데이터를 적재합니다.

  • 2

    AutoInvoice Master Program 실행

    AR → 요청 → AutoInvoice Master Program을 실행합니다. 처리 대상 일자 범위와 원천(Source)을 지정합니다.

  • 3

    오류 인보이스 검토 및 재처리

    AutoInvoice Exception Report를 조회하여 오류 건을 확인하고 수정 후 재실행합니다.

🏠 홈 › 2. 업무별 처리방법 › 2.3 수금 처리

💰 수금 처리

고객으로부터 수금된 금액을 Oracle EBS에 입력하고 인보이스에 적용하는 방법을 안내합니다.

AR› 수금› 수금 입력
💳
수동 수금 입력
  • 1

    수금 일자 및 금액 입력

    수금일, 통화, 수금 금액을 입력합니다. 통화가 외화인 경우 환율을 입력합니다.

  • 2

    수금 방법 선택

    계좌이체, 수표, 현금 등 수금 방법(Receipt Method)을 선택합니다.

  • 3

    고객 선택

    수금 고객을 검색하여 선택합니다. 고객 선택 후 해당 고객의 미결 인보이스가 조회됩니다.

  • 4

    인보이스 적용

    미결 인보이스 목록에서 해당 인보이스를 선택하고 적용 금액을 입력합니다.

🤖
자동 수금 (Lockbox)

은행 파일을 통해 수금을 자동으로 처리하는 방법입니다.

  • 1

    은행 파일 수신

    은행으로부터 수금 데이터 파일을 수신합니다. (고정 포맷 또는 BAI2 형식)

  • 2

    Lockbox 프로그램 실행

    AR → 요청 → Lockbox Transmission Program을 실행하고 파일을 업로드합니다.

  • 3

    오류 건 처리

    자동 매칭이 실패한 수금은 수동으로 인보이스를 지정하여 처리합니다.

❓
미적용 수금 처리
Unapplied / Unidentified Receipt 처리
⚠️
미적용 수금이란? 고객이 특정 인보이스를 지정하지 않고 입금하거나, 인보이스와 금액이 다른 경우 발생합니다. 미적용 수금은 매출채권 잔액에 포함되지 않으므로 반드시 처리해야 합니다.
수금 상태설명처리 방법
Unidentified고객을 특정할 수 없는 수금고객을 확인 후 고객 지정하여 Unapplied 처리
Unapplied고객은 알지만 인보이스 미지정해당 인보이스를 직접 선택하여 적용
On Account선수금으로 처리된 수금인보이스 발생 시 선수금 계정에서 전환

🔗 전표 대사

수금과 인보이스를 연결하여 잔액을 정리하고, 차이 조정 처리하는 방법을 안내합니다.

🔗
수금-인보이스 대사 절차
  • 1

    대사 대상 확인

    AR → 수금 조회에서 Unapplied 상태의 수금을 조회합니다. 인보이스 번호, 금액, 만기일을 기준으로 대사할 건을 선택합니다.

  • 2

    Apply 화면 진입

    수금 선택 후 [Apply] 버튼을 클릭합니다. 해당 고객의 미결 인보이스 목록이 표시됩니다.

  • 3

    인보이스 선택 및 금액 입력

    대사할 인보이스를 선택하고 적용 금액을 입력합니다. 수금 금액이 인보이스 금액보다 크면 나머지는 On Account 또는 다른 인보이스에 적용합니다.

  • 4

    조정(Adjustment) 처리

    소액 차이나 환율 차이가 있는 경우 Adjustment를 생성하여 잔액을 소멸시킵니다. 조정 금액은 설정된 한도 내에서만 처리 가능합니다.

💡
대사 취소 방법 잘못 적용된 대사는 수금 조회에서 해당 수금을 선택 후 [Reverse Application]을 클릭하여 취소할 수 있습니다. 취소 후 재대사를 진행합니다.
🏠 홈 › 2. 업무별 처리방법 › 2.5 신용 관리

⚖️ 신용 관리

고객 신용 한도 설정, 연체 관리, 독촉장 발행 등 AR 신용 관리 방법을 안내합니다.

💳
신용 한도 관리
  • 1

    신용 한도 설정

    고객 마스터 → [신용 프로파일] 탭에서 신용 한도 금액과 통화를 입력합니다.

  • 2

    한도 초과 처리 규칙 설정

    한도 초과 시 경고(Warning)만 표시할지, 오더 보류(Hold)를 걸지 결정합니다.

  • 3

    주기적 한도 검토

    Customer Credit Snapshot Report를 통해 고객별 신용 사용 현황을 주기적으로 검토합니다.

📬
독촉장 발행 (Dunning)
  • 1

    연체 기준 설정

    Dunning Letter 발행 조건(연체 일수, 금액)을 설정합니다. 1차(30일), 2차(60일), 3차(90일) 독촉장으로 단계적 발행이 가능합니다.

  • 2

    독촉장 생성 및 발송

    AR → 요청 → Print Dunning Letters 프로그램을 실행합니다. 이메일 또는 우편으로 발송합니다.

  • 3

    독촉 이력 관리

    고객별 독촉 발송 이력은 Customer Calls/Notes에 기록하여 추적합니다.

🏠 홈 › 2. 업무별 처리방법 › 2.6 마감 처리

📅 마감 처리

AR 월마감 절차와 GL 전송, 기간 마감 방법을 안내합니다.

📅
AR 월마감 체크리스트
순서대로 진행하세요
  • 1

    미완료 인보이스 처리

    Incomplete 상태의 인보이스를 조회하여 완료 처리합니다. AR → 전표 → 인보이스 조회에서 상태 = Incomplete로 필터링합니다.

  • 2

    미전기 트랜잭션 전기 처리

    General Ledger Transfer Program을 실행하여 AR 트랜잭션을 GL에 전기합니다. AR → 요청 → General Ledger Transfer.

  • 3

    미결 수금 확인

    Unapplied/Unidentified 수금을 전량 처리합니다. 처리 불가한 건은 상위자 승인 후 On Account 처리합니다.

  • 4

    AR-GL 잔액 조정

    AR Trial Balance Report와 GL 잔액을 대사합니다. 차이가 발생한 경우 원인을 파악하고 조정합니다.

  • 5

    기간 마감 (Close Period)

    AR → 설정 → 회계 기간에서 해당 기간의 상태를 Closed로 변경합니다. 마감 후에는 해당 기간에 새 트랜잭션 입력이 불가합니다.

⚠️
주의 기간 마감은 되돌릴 수 없습니다. AR-GL 잔액 대사가 완료된 후 마감을 진행하세요.

🏠 홈 › 4. 오류 처리 › 4.1 주요 오류 및 해결방법

🚨 주요 오류 및 해결방법

AR 업무 중 자주 발생하는 오류와 해결 방법을 안내합니다.

오류 메시지원인해결 방법
Period is not open 입력한 GL Date가 닫힌 회계 기간 AR → 설정 → 회계 기간에서 해당 기간을 Open으로 변경하거나, GL Date를 열린 기간으로 수정합니다.
AutoInvoice Exception: Invalid Account 인터페이스 데이터의 GL 계정이 잘못됨 RA_INTERFACE_LINES_ALL의 계정 정보를 확인하고, 유효한 계정으로 수정 후 재실행합니다.
Customer on Credit Hold 고객의 신용 한도 초과 또는 수동 보류 처리 고객 마스터 → [신용 프로파일]에서 Hold 해제하거나 신용 한도를 상향합니다.
Receipt cannot be reversed 이미 적용된 수금을 직접 삭제하려 시도 [Apply] → [Reverse Application]으로 대사를 먼저 취소한 후 수금을 역분개합니다.
GL Transfer Incomplete AR-GL 전송 중 오류 발생 AR → 요청 → 출력에서 오류 로그를 확인하고, 원인 해결 후 재실행합니다.
🚨
오류 해결이 어려운 경우 시스템 관리자 또는 Oracle Support에 SR(Service Request)을 등록하여 문의하세요. 오류 메시지, 발생 시간, 관련 트랜잭션 번호를 함께 제공하면 빠른 해결이 가능합니다.
